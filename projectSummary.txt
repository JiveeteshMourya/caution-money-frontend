--------------------------------------------------------------------------------
YEH APP HAI KYA BHAI?
--------------------------------------------------------------------------------

Toh basically — college mein join karte waqt ₹5,000 jama karte ho na?
Woh "caution money" hota hai. Ek tarah ka hostage money hai tumhara, college
ke paas. Jab graduation hoti hai, toh woh paisa wapas milna chahiye.

Lekin pehle kya hota tha?

  - Library jaao, stamp lo
  - Sports room jaao, stamp lo
  - Hostel warden ke paas jaao, unhe dhundo pehle
  - Department ke HOD ko pakdo
  - Accounts mein line mein khade raho
  - Ek mahine baad: "Form galat bhara tha, dubara aao"

Sab drama. Sab chakkar. Zero paisa.

Ab kya hota hai is app se?

  - Online form bharo
  - Har department wala apne laptop se approve karta hai
  - Accounts process karta hai
  - Paisa aata hai
  - Tu ghar pe baitha Netflix dekh raha hai

Yahi hai yeh app. Simple. Seedha. No nautanki.

--------------------------------------------------------------------------------
KON USE KARTA HAI? (The Cast)
--------------------------------------------------------------------------------

  1. STUDENTS (The Heroes / Victims)
     - Register karo, application bharo, bank details dalo
     - Phir bas track karte raho — kaun clear hua, kaun pending hai
     - Jab sab ho jaaye, status page pe "Refund Processed" dekho
       aur ek choti si khushi feel karo

  2. ADMIN WALE (The Gatekeepers)
     - Library staff   → library clearance approve karo
     - Sports staff    → sports clearance (probably sabse fast, koi dues nahi hote)
     - Hostel warden   → hostel clearance (sabse suspicious wale)
     - Department HOD  → academic clearance
     - Accounts staff  → final boss — yeh approve kare toh paisa aaye
     - Superadmin      → God mode. Sab kuch dekh sakta hai, sab kuch kar sakta hai.
                         Basically IT wala bhaiya.

--------------------------------------------------------------------------------
APP KA FLOW — STEP BY STEP (Ek Baar Samjho)
--------------------------------------------------------------------------------

Step 1: Register karo
  → /student/login pe jaao → form bharo → account ban gaya → done

Step 2: Application Submit karo
  → Apni details + bank account number daalo → Submit maaro
  → Ab baksh do khud ko, baaki system karega

Step 3: Departments mein chakkar
  → Har department wala apni screen pe dekh raha hai teri application
  → Wo ya toh "Cleared" maarte hain ya "On Hold" (matlab kuch gadbad hai)
  → 5 departments hain: Library, Sports, Hostel, Dept, Accounts

Step 4: Sab clear ho gaye
  → Pehle 4 departments clear kar dein → status banta hai "Fully Cleared"
  → Ab accounts ka wait karo

Step 5: Accounts wala process karta hai
  → Accounts login karta hai, button dabata hai, transaction ID generate hoti hai
  → Status: "Refund Processed" ✓

Step 6: Tu khush
  → Dashboard kholo → dekho → paisa aaya ya nahi woh toh bank batayega
  → App ka kaam ho gaya

PAGES — KYA KAHAN HAI
--------------------------------------------------------------------------------

PUBLIC (koi bhi dekh sakta hai, login ki zarurat nahi):
  - Landing Page (/)      → "Welcome to the portal" wala intro page
                            Agar pehle se logged in ho toh seedha dashboard

STUDENT KE PAGES (sirf logged-in students):
  - Login/Register page   → nayi life shuru karo
  - Student Dashboard     → main hub — status, clearances, sab kuch ek jagah
  - Student Application   → form bharo / edit karo
  - Student Status        → step-by-step timeline — kahan tak pahuncha refund

ADMIN KE PAGES (staff wale log):
  - Admin Login           → alag login, student wala nahi chalega
  - Admin Dashboard       → numbers dekho — kitne applications, kitne clear, etc.
  - Admin Applications    → poori list, search kar sakte ho, filter kar sakte ho
  - Application Detail    → ek student ki poori application, action lo
  - Admin Clearances      → apne department ki clearances approve/hold karo
  - Admin Refunds         → sirf accounts + superadmin ke liye, final processing

--------------------------------------------------------------------------------
FOLDERS KA SCENE (Ghar ka structure)
--------------------------------------------------------------------------------

src/ ke andar yeh hain:

  pages/         → Jo pages dikhte hain screen pe, woh sab yahan
    student/     → Student wale pages
    admin/       → Admin wale pages

  components/    → Chhote chhote reusable cheezein — buttons, cards, modals
    common/      → Jo cheezein kai pages pe use hoti hain (jaise status badge)

  services/      → Yeh files backend se baat karti hain (API calls)
                   "Bhai server, yeh data de" — yahi kaam inका

  hooks/         → Data lao, manage karo — reusable logic
                   Page sirf display karta hai, hook data laata hai

  context/       → Global variable jaisi cheez — "kaun logged in hai?"
                   Poori app isko read kar sakti hai

  router/        → Kaun sa URL pe kaun sa page dikhega, yeh decide karta hai
                   + Protected routes: "Teri aukat hai is page pe jaane ki?"

  config/        → Hardcoded values — department names, status labels, amounts
                   Ek jagah change karo, poori app mein change ho jaata hai

  utils/         → Chote helper functions — date format karo, account mask karo

--------------------------------------------------------------------------------
LIBRARIES — KYA USE KI AUR KYU (The Squad)
--------------------------------------------------------------------------------

REACT
  Kya hai:  UI banane ki main library
  Kyun:     Poori app ko chhote chhote "components" mein todta hai.
            Ek <ClearanceCard /> component likho, kahin bhi use karo.
            Ek giant HTML file mein sab likhne ki zarurat nahi.
            Basically LEGO blocks hai, HTML nahi.

REACT ROUTER DOM
  Kya hai:  Page navigation bina browser reload ke
  Kyun:     Jab tu "Dashboard" pe click karta hai, page reload nahi hota.
            Sirf content swap hota hai — isliye app fast lagti hai.
            Aur protected routes: agar tu admin nahi hai toh
            /admin/dashboard pe jaane ki koshish karo — seedha kick out.

AXIOS
  Kya hai:  Backend se baat karne ka tool (HTTP requests)
  Kyun:     Jab application submit hoti hai, axios woh data server ko bhejta hai.
            Jab page load hota hai, axios server se fresh data laata hai.
            Aur ek baar setup karo — automatically login token lagate rehta hai
            har request mein. Manual kaam nahi.

DATE-FNS
  Kya hai:  Dates ke saath kaam karne ki library
  Kyun:     Server se date aati hai aise: "2026-03-26T10:30:00Z"
            Yeh library ise aise dikhata hai: "26 Mar 2026, 10:30 AM"
            Ya "2 hours ago". Yeh logic khud likhna padta toh rona aata.

REACT HOT TOAST
  Kya hai:  Wo chhoti popup notification jo screen ke upar aati hai
  Kyun:     "Application submitted!" ya "Clearance approved!" — user ko
            instant feedback chahiye. Yeh library automatically
            ek toast dikhata hai. Blink karo, gaayab.

REACT ICONS
  Kya hai:  Ready-made icons ka bhandaar
  Kyun:     Image files download karne ki zarurat nahi. Bas import karo
            aur <FiBook /> ya <FiCheckCircle /> likh do. Hundreds of icons,
            zero headache.

VITE
  Kya hai:  Development server + build tool
  Kyun:     localhost:5173 pe app chalaata hai. React ka code jo hum likhte hain
            woh browser directly nahi samajhta — Vite usse compile karta hai.
            Create React App se kaafi fast hai. Jab save karo, 0.1 second mein
            browser update hota hai. No cap.

--------------------------------------------------------------------------------
LOGIN AUR SECURITY — THODA SAMJHO
--------------------------------------------------------------------------------

Jab login karo:
  → Server ek "token" deta hai — ek lamba string, jaise ID card
  → App ise browser ke localStorage mein save kar leta hai
  → Har API request ke saath yeh token automatically jaata hai
  → Server dekhta hai: "Haan, yeh banda authorized hai"
  → Agar token expire ho gaya ya invalid hai → app seedha logout kar deta hai
    aur home page pe bhej deta hai. No mercy.

AuthContext — Yeh kya hai?
  → Ek global "kaun logged in hai" wali cheez
  → Koi bhi page ise padh sakta hai: user kaun hai, role kya hai, loading hai ya nahi
  → Ek baar login karo, poori app ko pata chal jaata hai

Protected Routes:
  → Student /admin/dashboard pe jaane ki koshish kare → BOUNCE
  → Admin /student/dashboard pe jaaye → BOUNCE
  → Library staff /admin/refunds pe jaaye → BOUNCE (sirf accounts + superadmin)
  → Sab apni-apni lane mein rehte hain

--------------------------------------------------------------------------------
DATA KAISE AATA HAI — HOOKS KA SCENE
--------------------------------------------------------------------------------

Har page apna data khud nahi fetch karta. Kaam hooks karte hain.
Hooks = reusable data fetchers. Ek baar likho, kahin bhi use karo.

  useMyApplication()     → "Meri application de"
  useApplicationList()   → "Saari applications de, filter ke saath"
  useDashboardStats()    → "Admin ke numbers de"
  useApplicationDetail() → "Is specific application ki poori detail de"

Page bas itna karta hai:
  const { application, loading } = useMyApplication()

Baaki sab hook handle karta hai — data fetch karo, loading dikhao, error lo.
Page sirf render karta hai. Clean division of responsibility.

--------------------------------------------------------------------------------
BACKEND SE KAISE BAAT HOTI HAI
--------------------------------------------------------------------------------

Sab API calls /api/... pe jaati hain.
Development mein: yeh localhost:5000 pe forward hota hai (Vite ka proxy)
Production mein: Vercel pe deploy hai, backend URL environment variable se aata hai

Ek single Axios instance hai (utils/api.js) jo:
  → Base URL set karta hai
  → Automatically token lagata hai har request mein
  → 401 error aaye toh khud logout karke ghar bhej deta hai

Koi bhi service file seedha axios use nahi karti — sirf is instance ko use karti hai.
Isliye token logic ek hi jagah hai, baar baar nahi likhna.

--------------------------------------------------------------------------------
DEPLOYMENT — APP KAHAAN JAATI HAI
--------------------------------------------------------------------------------

App Vercel pe deploy hai — free hosting, ek push mein live.

Ek cheez important: yeh "Single Page App" hai. Matlab browser /student/dashboard
seedha khole toh Vercel confuse ho jaata — "yeh file kahan hai?"
vercel.json mein ek rewrite rule hai jo sab URLs ko index.html pe bhejti hai
aur React apna routing handle karta hai. Without this — 404 milta. Iska dhyan rakhna.
