  CAUTION MONEY REFUND PORTAL – FRONTEND TEAM RESPONSIBILITIES SUMMARY

--------------------------------------------------------------------------------
1. JIVEETESH  |  Tech Lead & Core Architecture
--------------------------------------------------------------------------------
Role:
  - Led all technical research, ideation, and architectural decisions
  - Established the project structure, module boundaries, and development standards

Responsibilities:
  - Defined the full folder architecture: pages/, services/, hooks/, context/,
    router/, config/, utils/ with @/ alias resolution
  - Configured Vite build tool (vite.config.js): React plugin, path alias,
    dev server (port 5173), and API proxy to backend (localhost:5000)
  - Designed and implemented the authentication system:
      * AuthContext.jsx — React Context providing user state, login(), logout(),
        and loadUser() with localStorage persistence (iehe_token, iehe_user)
      * RequireStudent.jsx — Route guard for student-protected pages
      * RequireAdmin.jsx  — Route guard for admin-protected pages with
        role-based access control (6 roles: superadmin, library, sports,
        hostel, department, accounts)
  - Built the API client layer (src/utils/api.js):
      * Axios instance with /api baseURL
      * Auto-attach Bearer token from localStorage
      * 15s timeout configuration
      * Auto-logout on 401 Unauthorized responses
      * Request/response interceptors
  - Implemented all service modules:
      * authService.js — studentLogin, studentRegister, adminLogin, getMe
      * applicationService.js — getMyApplication, submitApplication,
        getAllApplications, getApplicationById, updateClearance,
        processRefund, getDashboardStats
  - Authored the application router (AppRoutes.jsx): all public, student, and
    admin routes with role-based access enforcement
  - Defined all global constants (config/constants.js):
      * REFUND_AMOUNT, PASSOUT_YEARS, DEPARTMENTS, role labels/icons,
        clearance role maps, application status steps, storage keys
  - Built utility functions:
      * formatters.js — formatDate, formatDateTime, timeAgo, getInitials,
        maskAccount
      * mappers.js — statusLabel, statusPill, clearanceLabel
  - Enforced coding standards: Prettier config (.prettierrc), npm scripts

Key Files:
  vite.config.js, src/context/AuthContext.jsx,
  src/router/AppRoutes.jsx, src/router/RequireStudent.jsx,
  src/router/RequireAdmin.jsx, src/utils/api.js,
  src/services/authService.js, src/services/applicationService.js,
  src/config/constants.js, src/utils/formatters.js, src/utils/mappers.js,
  src/App.jsx, src/main.jsx, .prettierrc

4. SHYAMHARI  |  Student Application & Status Developer
--------------------------------------------------------------------------------
Role:
  - Developed the student application submission and status tracking features

Responsibilities:
  - Built StudentApplication.jsx — 4-step refund application form:
      * Step 1 (Personal Info): name, enrollment, year, department (pre-filled,
        editable) with client-side validation
      * Step 2 (Bank Details): account number, IFSC code, account holder name,
        bank name — with input masking and format validation

        terms and conditions acknowledgment

      * Guard to prevent duplicate submissions (redirect if application exists)
  - Built StudentStatus.jsx — visual application progress tracker:
      * Step-by-step progress visualization across all 6 statuses:
        Submitted → Under Review → Partially Cleared → Fully Cleared →
        Refund Processed (with On Hold variant)
      * Department clearance status grid (Library, Sports, Hostel, Department)
      * Hold reason display when application is on hold
      * Timeline component showing activity history with timestamps
      * Refund transaction details when refund is processed

Key Files:
  src/pages/student/StudentApplication.jsx,
  src/pages/student/StudentStatus.jsx
