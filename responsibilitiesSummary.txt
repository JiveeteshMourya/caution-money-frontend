1. JIVEETESH  |  Tech Lead, Head Nerd, "Bhai ye kaise karte hain" wala banda
--------------------------------------------------------------------------------
Vibe:
  - Wo banda jo 2 baje raat ko bhi architecture diagrams bana raha tha
  - Literally poore project ka baap. Research ki, socha, banaya, sab ko samjhaya
  - "Pehle soch lo, phir code karo" — ye sirf usi ne follow kiya

Kya kiya usne:
  - Poora folder structure design kiya — pages/, services/, hooks/, context/,
    router/, config/, utils/ — ek dum systematic, bilkul apni life ke opposite
  - Vite configure kiya (vite.config.js): React plugin, @/ alias, dev server
    port 5173 pe, aur backend ka proxy localhost:5000 pe
    (matlab frontend ko pata hi nahi backend kahan hai, bus kaam karta hai)
  - Auth system khada kiya, akela:
      * AuthContext.jsx — React Context jo user ko yaad rakhta hai login ke baad
        (localStorage mein iehe_token aur iehe_user daal ke, jugaad but works)
      * RequireStudent.jsx — "Aye bhai tu student hai? Nahi? Nikal yahan se"
      * RequireAdmin.jsx  — 6 roles handle karta hai: superadmin, library,
        sports, hostel, department, accounts
        (ek ek department ka bouncer basically)
  - API client banaya (utils/api.js):
      * Axios ka setup, /api baseURL
      * Token automatically lagate hain har request mein, manually nahi karna
      * 15 second mein timeout — library wale uncle se zyada patient nahi hai
      * 401 aaya? Auto logout. Seedha. No drama.
  - Dono services likhe:
      * authService.js — login, register, adminLogin, getMe
      * applicationService.js — getMyApplication, submitApplication,
        getAllApplications, getApplicationById, updateClearance,
        processRefund, getDashboardStats
        (basically poori app ki backbone, credit dena chahiye tha)
  - Router likha (AppRoutes.jsx) — kaun sa route kahan jaata hai, role check,
    guards lagaye, sab kuch. GPS tha poori app ka.
  - Constants define ki (config/constants.js):
      * REFUND_AMOUNT = 5000 (wahi 5k jo 4 saal se college ne rok rakha tha)
      * DEPARTMENTS, PASSOUT_YEARS, role labels, status steps — sab
  - Utility functions:
      * formatters.js — date format, "2 days ago" wala time, account number mask
      * mappers.js — status ko readable label mein convert karo
  - Prettier config lagaya taaki kisi ka bhi code ganda na dikhे
    (kuch logo ka dikhta tha, tab bhi)

Key Files:
  vite.config.js, src/context/AuthContext.jsx,
  src/router/AppRoutes.jsx, src/router/RequireStudent.jsx,
  src/router/RequireAdmin.jsx, src/utils/api.js,
  src/services/authService.js, src/services/applicationService.js,
  src/config/constants.js, src/utils/formatters.js, src/utils/mappers.js,
  src/App.jsx, src/main.jsx, .prettierrc

4. SHYAMHARI  |  Application Form & Status Developer, "Form bharna easy karo"
--------------------------------------------------------------------------------
Vibe:
  - 4 step form banaya usne — clearly zyada patient hai baaki logo se
  - Status page itna acha banaya ki anxiety kam ho jaati hai thodi

Kya kiya usne:
  - StudentApplication.jsx — 4 step ka mega form:
      * Step 1 (Personal Info): naam, enrollment, year, department pehle se
        filled hain (Sulabh ka data use hua), edit bhi kar sakte ho
      * Step 2 (Bank Details): account number, IFSC, bank naam — sab validate
        hota hai, galat IFSC dalo toh bata dega
        (SBI wale dhyan se bharna, 11 digit hota hai)
      * Step 3 (Declaration): checkbox tick karo ki "haan bhai sab sach hai"
        — digital signature basically, jugaad version
      * Step 4 (Submit): poora review karo, account number masked dikhega
        (****1234 — security conscious hai ye app)
      * Double submission? Nahi hoga. Already applied hai? Redirect kar dega.
        (ek baar kafi hai, college ko bhi pata hai)
  - StudentStatus.jsx — progress tracker, anxiety ka ilaaj:
      * 6 stages ka visual progress:
        Submitted → Under Review → Partially Cleared → Fully Cleared →
        Refund Processed (aur On Hold — jab library wale naaraaz ho)
      * 4 departments ki clearance grid — green tick = khushi, red = tension
      * On Hold? Reason bhi dikhega — "book return nahi ki" etc.
      * Timeline — kab kya hua, poori history
      * Refund ho gaya? Transaction details bhi dikhega — kitne aaye, kab aaye

Key Files:
  src/pages/student/StudentApplication.jsx,
  src/pages/student/StudentStatus.jsx
